## Scope 1: Foundation

**Branch:** `scope/foundation`
**Status:** `[DONE]`
**Depends on:** *(none)*

> This scope builds the entire backend — config loading, Jira API client, and recursive fetcher. It has no UI. It must be fully functional and testable in isolation before Scope 2 begins.

### Context for Claude Code
- Read `P-0001/architecture.md` for module layout and file structure
- Read `P-0001/data_model.md` for all dataclasses (`Config`, `Ticket`, `TicketTree`, `Comment`, `ReportConfig`)
- Read `P-0001/workflows.md` diagrams WF-01 and WF-02 for fetch logic

### Tasks
- [x] Create project structure: `app.py`, `core/__init__.py`, `core/config.py`, `core/models.py`, `core/jira_client.py`, `core/fetcher.py`, `core/gantt.py`, `core/excel_builder.py`
- [x] Create `requirements.txt` with: `streamlit`, `requests`, `openpyxl`, `python-dotenv`
- [x] Create `.env.example` with all required keys documented:
  ```
  JIRA_URL=https://your-org.atlassian.net
  JIRA_USER=your-email@example.com
  JIRA_TOKEN=your-api-token
  JIRA_START_DATE_FIELD=customfield_XXXXX
  JIRA_DUE_DATE_FIELD=duedate
  JIRA_PROJECT_TYPE=nextgen   # or: classic
  ```
- [x] Implement `core/models.py`: all dataclasses as specified in `data_model.md`
- [x] Implement `core/config.py`: `Config.from_env()` — load `.env`, validate all required vars, raise `ConfigError` with list of missing keys if any are absent
- [x] Implement `core/jira_client.py`: `JiraClient` class with:
  - `get_issue(ticket_id)` → raw dict
  - `get_children(parent_id, project_type)` → list of raw dicts (JQL, paginated, max_results=100)
  - `get_comments(ticket_id)` → list of raw comment dicts (paginated, 50/page)
  - Exponential back-off retry on HTTP 429 (max 3 retries, delays: 1s, 2s, 4s)
  - Raise `AuthError` on 401/403, `TicketNotFoundError` on 404, `JiraAPIError` on other non-2xx
- [x] Implement `core/fetcher.py`:
  - `fetch_tree(ticket_id, config, client)` → `TicketTree`
  - Recursive to max depth 2 (depth 0 = Feature, 1 = Epic, 2 = Story/Task)
  - Parse `start_date` from `config.start_date_field_id`, `due_date` from `duedate` field
  - Parse assignee from `assignee.displayName` (null-safe)
  - `fetch_comments(tree, config, client)` → populates `tree.comments` (see WF-03)
- [x] Implement `core/gantt.py`:
  - `calc_date_range(tickets)` → `(min_date, max_date)` ignoring None values
  - `generate_week_columns(min_date, max_date)` → list of `date` objects (Mondays)
  - `ticket_in_week(ticket, week_monday)` → `bool`
- [x] Write a standalone test script `test_fetch.py` (not a formal test suite — a runnable script that calls `fetch_tree` with a real ticket ID and prints a summary of results). This validates the Jira connection before Scope 2 begins.

### Acceptance Criteria
- **cap-001:** App starts without error when `.env` contains all required variables. App shows a clear error message listing missing variables if any are absent.
- **cap-002:** Authenticated requests succeed. HTTP 401/403 surfaces a user-readable error. HTTP 429 is retried with exponential back-off (max 3 retries).
- **cap-003:** Given a Feature ticket ID, the fetcher returns the Feature + all Epics + all Stories/Tasks. Correctly handles Epics with zero children and Features with zero Epics. Depth is capped at 3 levels; no infinite recursion.

## Scope 2: Excel Report

**Branch:** `scope/excel-report`
**Status:** `[DONE]`
**Depends on:** Scope 1 (`TicketTree`, `Comment`, `Config`, `gantt.py` must exist)

> This scope builds the entire Excel output — all five sheets — without any Streamlit UI. The builder should be callable as a standalone function that accepts a `TicketTree` and returns bytes.

### Context for Claude Code
- Read `P-0001/spec.md` Acceptance Criteria sections for cap-005 through cap-010
- Read `P-0001/wireframes.md` for the layout of each Excel sheet
- Read `P-0001/workflows.md` diagram WF-04 for sheet construction order and WF-05 for Gantt week-column logic

### Tasks
- [x] Implement `core/excel_builder.py` — `build_excel(tree: TicketTree, config: ReportConfig) -> bytes`:
  - `_sheet_tickets(wb, tree)` — flat DFS list, all fields, hyperlinks in Link column, auto-column width, freeze top row
  - `_sheet_simplified_gantt(wb, tree, config)` — open epics only, sorted by due date (None last), week columns, coloured cell fill for date ranges
  - `_sheet_full_gantt(wb, tree, config)` — same as simplified gantt + child sub-rows after each epic; sub-rows indented in ID column; no date fill for children
  - `_sheet_updates(wb, tree, config)` — comments on Epics + children, last 7 days, sorted by date desc
  - `_sheet_weekly_updates(wb, tree, config)` — root Feature comments only, last 7 days, sorted by date desc
- [x] Gantt cell fill: use a single solid fill colour (e.g. light blue `00B0F0`) for in-range cells; plain white for out-of-range
- [x] Gantt week column headers: format as `DD/MM` of each Monday
- [x] Gantt sub-rows (Full Gantt): prefix ID cell value with two spaces for visual indent; do NOT merge cells
- [x] Handle edge case: no dated open epics → write sheet with header row + message "No open epics with dates found"
- [x] Handle edge case: no comments in last 7 days → write sheet with header row + message "No comments in the last 7 days"
- [x] `build_excel` saves output to `./output/YYYYMMDD_{ticket_id}.xlsx` (create dir if absent) and returns the same bytes
- [x] Write a standalone test script `test_excel.py` that loads a saved `TicketTree` fixture (or calls Scope 1 live) and generates a report to `./output/test_output.xlsx` for manual inspection

### Acceptance Criteria
- **cap-005:** Tickets sheet has one row per ticket, all columns present, null dates blank, Link column contains clickable hyperlinks.
- **cap-006:** Simplified Gantt shows only open-status epics, ordered by due date ascending (nulls at bottom), week columns auto-span earliest start to latest due, cells filled within each ticket's date range.
- **cap-007:** Full Gantt matches Simplified Gantt epics + shows first-level children as indented sub-rows with ID, Name, Assignee, Status; no date fill on sub-rows.
- **cap-008:** Updates sheet lists comments from Epics and children only, from last 7 days, sorted by date descending.
- **cap-009:** Weekly Updates sheet lists comments from root Feature ticket only, from last 7 days, sorted by date descending.
- **cap-010:** `build_excel` returns xlsx bytes and simultaneously writes `./output/YYYYMMDD_<ticket-id>.xlsx`; `./output/` is created if absent.

## Scope 3: Streamlit UI

**Branch:** `scope/streamlit-ui`
**Status:** `[DONE]`
**Depends on:** Scope 1 and Scope 2 (all core modules must be functional)

> This scope wires the UI to the backend. `app.py` contains only UI logic — no business logic. All heavy lifting is delegated to `core/`.

### Context for Claude Code
- Read `P-0001/wireframes.md` for all three Streamlit screen states (idle, generating, report ready, error)
- Read `P-0001/workflows.md` diagram WF-01 for startup flow

### Tasks
- [x] Implement `app.py`:
  - On startup: call `Config.from_env()`. If `ConfigError` raised, render `st.error` with missing variable list and `st.stop()`
  - Sidebar: `st.multiselect` for open statuses, pre-populated with `["To Do", "In Progress", "In Review"]`
  - Main area: `st.text_input` for ticket ID, `st.button("Generate Report")`
  - On button click: show `st.spinner` / `st.progress` while calling `fetch_tree` and `fetch_comments`
  - On success: show summary metrics (epic count, story count, comment count, saved file path) and `st.download_button` with xlsx bytes
  - On any raised exception: catch and render `st.error` with a user-friendly message (not a stack trace)
  - Do NOT use `st.experimental_rerun` or session state beyond what is needed to hold the generated report bytes
- [x] Confirm `streamlit run app.py` starts cleanly with a valid `.env`
- [x] Confirm error states render correctly: missing `.env` var, bad ticket ID, auth failure

### Acceptance Criteria
- **cap-004:** User can enter a ticket ID, configure open statuses in the sidebar, click Generate, see progress feedback, and receive a download button upon success. Errors are shown inline without stack traces.
- **cap-010** (UI side): Download button triggers browser file download of the xlsx. File is also present in `./output/` after generation.
